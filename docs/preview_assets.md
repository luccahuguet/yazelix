# Preview Assets

Yazelix keeps the in-repo preview surface intentionally small:

- `assets/logo.png`
- `assets/screenshots/yazelix_current_example.png`

The repository should keep only one current static screenshot unless a second asset defends a clear docs need. If Yazelix later gets a separate public website or another real front-door surface, richer demos can be added deliberately there instead of bloating the repo by default.

## Capture Checklist

1. Start from a fresh Yazelix session with the current default visual stack.
2. Show the core workspace shape clearly: sidebar visible, managed editor visible, bottom terminal visible.
3. Capture a real project/worktree so the screenshot looks like normal use, not an empty mockup.
4. Save the maintained README screenshot to `assets/screenshots/yazelix_current_example.png`.
5. If a new asset is added, document why one screenshot is not enough and remove any superseded asset from the repo head.
